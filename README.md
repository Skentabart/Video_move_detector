# Сводный кадр из видео (карта движения)

Этот инструмент помогает “сжать” длинную запись с камеры наблюдения в **одну картинку**, на которой видно, **где и как часто было движение** (чтобы быстро понять: подходил ли кто‑то).

## Как работает

- Берётся кадр раз в N секунд (по умолчанию: **1 кадр/сек**).
- Используется вычитание фона (MOG2), чтобы выделить движущиеся объекты.
- Два режима:
  - **once**: красным показываются места, где движение было **хотя бы один раз** (идеально для “подходил ли сотрудник к столу один раз”).
  - **heat**: “частота” движения (чем чаще — тем заметнее).
- Синим рисуются **следы (траектории)** по центрам движущихся объектов — помогает увидеть разовый подход как линию/пятно.
- Опционально можно включить **Фотофиксацию входа человека в выбранную зону**:
  - Используется детектор людей (**YOLO** если установлен, иначе **HOG**).
  - Каждое событие “человек вошёл в зону” сохраняется как отдельное фото + `entries.csv`.

## Установка

В папке `Video/`:

```bash
python -m pip install -r requirements.txt
```

### (Опционально) Точный детектор людей YOLO

Если включаете в настройках `Детектор: yolo`, установите:

```bash
python -m pip install ultralytics
```

## Как найти подход к объекту (типовой сценарий)

1) Выберите видео.
2) Нажмите **«Выбрать зону…»** и выделите объект прямоугольником.
3) Включите **«Фотофиксация входа человека в зону»**.
4) Нажмите **«Старт»**.

После обработки появится папка `*_entries/` с:
- фото каждого входа (на фото подписан `ID`)
- `entries.csv` (лог событий)

Чтобы отфильтровать **конкретного сотрудника**:
- посмотрите `ID` нужного сотрудника на фото
- задайте этот `ID` в поле **«ID сотрудника для фиксации (-1 = все)»**
- прогоните видео снова — сохранит только входы этого ID

## Запуск

```bash
python video_summary_app.py
```

Дальше:

- Нажмите **«Выбрать…»** и выберите видео в проводнике.
- Нажмите **«Старт»**.
- Итог будет сохранён рядом с видео как `*_motion_summary.png`.

## Настройки (рекомендации для 9 часов / ~1ГБ)

- **Обрабатывать кадр каждые (сек)**:
  - 1.0 — максимум деталей, дольше по времени
  - 2.0–5.0 — обычно достаточно, заметно быстрее
  - 10.0 — очень быстро, но можно пропустить короткие события
- **Порог движения**:
  - 16–40 — это `varThreshold` для MOG2 (зависит от шума/качества)
- **Мин. площадь объекта**:
  - если не видит человека — уменьшайте (например 150–300)
  - если ловит шум/тени — увеличивайте
